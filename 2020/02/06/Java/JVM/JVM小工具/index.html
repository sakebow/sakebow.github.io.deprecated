一点点拓展

在这里不得不提一下SSL链接、HTTP、HTTPS的关系[1]。

首先是HTTP，它只有三个步骤。

  • 封装数据
  • 使用TCP/IP传输
  • 服务器接收

这三个步骤就可以正常访问一个超文本链接，也就是我们所说的网页。

然后是HTTPS，它比HTTP多一个S，也就是SSL安全认证，它包含了四个步骤：

  • 封装数据
  • 使用SSL加密
  • 使用TCP/IP加密
  • 服务器接收

其中由于SSL加密不可避免地加长了每一段数据帧的字长，所以也无可避免的增加了传输消耗，延长了响应时间。但是这些消耗带来了较为安全的传输方式，在可接受范围内。
